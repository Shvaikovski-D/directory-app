### 1. Краткое введение  
Проблема: при переключении строки таблицы в режим редактирования поля ввода увеличивают высоту ячеек, строка «выпирает» за границы таблицы и часть содержимого становится невидимой. Это типичная ситуация: **стандартные Material поля и инпуты имеют внутренние отступы и поведение, которое меняет высоту строки**, если не ограничить их стилями и поведение таблицы.

**Из документа:** в компоненте задана высота строки `.data-row { height: 60px; }`.   
**Из документа:** поля ввода в режиме редактирования находятся внутри `mat-form-field` с классом `edit-field`. 

---

### 2. Пошаговое объяснение ключевых концепций (почему так происходит и что менять)

1. **Мат‑поля и инпуты увеличивают высоту**  
   `mat-form-field` и внутренние элементы (wrapper, infix, label, hint, error) добавляют отступы и могут менять высоту строки. Если строка таблицы имеет фиксированную высоту, но содержимое выше — браузер увеличит высоту строки.

2. **Таблица и поведение ячеек**  
   По умолчанию таблица может менять ширину/высоту ячеек в зависимости от содержимого. Свойство `table-layout: auto` позволяет содержимому влиять на ширину столбцов. Для стабильного расположения лучше использовать `table-layout: fixed` и фиксированные ширины столбцов.

3. **Box sizing и вертикальное выравнивание**  
   Нужно контролировать `box-sizing`, `height` и `line-height` инпутов, а также выравнивание внутри `mat-form-field` (сделать infix flex‑контейнером и центрировать по вертикали).

4. **Ошибки валидации и mat-error**  
   Появление `<mat-error>` увеличивает высоту. Нужно либо показывать ошибки в другом месте (tooltip, snackbar), либо резервировать место под ошибки (фиксированная высота для строки ошибок), либо показывать ошибки только при раскрытии (dialog).

5. **Подходы к решению**  
   - **Ограничить высоту строки** и сделать поля/контейнеры адаптированными под эту высоту.  
   - **Сделать `table-layout: fixed` и задать ширины столбцов** — это предотвратит горизонтальное «выпирание».  
   - **Стилизовать `mat-form-field` и input`ы**: убрать лишние паддинги, сделать infix flex и задать фиксированную высоту инпута.  
   - **Альтернатива**: редактирование в модальном окне или в плавающем overlay, чтобы не ломать layout таблицы.

### 3. Мини‑пример кода с пояснениями  
Ниже — минимальные правки в CSS и небольшие шаблонные правки, которые сохранят высоту строки 60px и не позволят полям «выталкивать» строку.

#### CSS (добавить/заменить в компоненте)
```css
/* 1) Жёсткая фиксация layout таблицы */
.forklifts-table {
  width: 100%;
  border-collapse: collapse;
  table-layout: fixed; /* важно: предотвращает перерасчёт ширин по содержимому */
}

/* 2) Фиксированная высота строки (уже есть) */
.data-row {
  height: 60px;
  max-height: 60px;
  overflow: hidden; /* на случай, если что-то всё же больше */
}

/* 3) Подогнать mat-form-field под высоту строки */
.data-row .mat-form-field,
.data-row .edit-field {
  height: 100%;
  margin: 0;
  box-sizing: border-box;
}

/* 4) Сделать infix flex-контейнером и центрировать содержимое по вертикали */
.data-row .mat-mdc-form-field-infix {
  display: flex;
  align-items: center;
  height: 100%;
  padding: 0 0.25rem; /* минимальные отступы */
}

/* 5) Ограничить высоту самого input и убрать лишние внутренние отступы */
.data-row input.mat-mdc-input-element {
  height: 1.6rem; /* или calc(100% - 12px) при необходимости */
  line-height: 1.6rem;
  padding: 0.15rem 0.25rem;
  box-sizing: border-box;
  overflow: hidden;
  white-space: nowrap;
  text-overflow: ellipsis;
}

/* 6) Для числового поля убрать стрелки и подогнать стиль */
.data-row input[type="number"]::-webkit-outer-spin-button,
.data-row input[type="number"]::-webkit-inner-spin-button {
  -webkit-appearance: none;
  margin: 0;
}

/* 7) Резерв места для mat-error (чтобы появление ошибки не ломало высоту) */
.data-row .mat-error {
  position: absolute;
  transform: translateY(100%); /* показывать под строкой, не меняя её высоту */
  background: transparent;
  font-size: 0.8rem;
  white-space: nowrap;
}

/* 8) Убедиться, что ячейки не растягиваются по содержимому */
.forklifts-table td, .forklifts-table th {
  overflow: hidden;
  text-overflow: ellipsis;
  white-space: nowrap;
}
```

#### Шаблон (необязательно, но полезно)  
Обёрнуть содержимое ячейки в контейнер, чтобы чётко разделить режимы просмотра/редактирования:
```html
<td mat-cell *matCellDef="let element; let i = index" class="column-brand">
  <div class="cell-view" *ngIf="!isEditing(element.id)">{{ element.brand }}</div>

  <div class="cell-edit" *ngIf="isEditing(element.id)">
    <mat-form-field appearance="outline" class="edit-field">
      <input matInput [formControl]="getBrandControl(i)!" placeholder="Марка" />
    </mat-form-field>
  </div>
</td>
```
**Пояснение:** контейнер `.cell-edit` + стили выше гарантируют, что `mat-form-field` занимает ровно высоту строки и не добавляет дополнительных отступов.

### 4. Возможные ошибки новичков и как их избежать

- **Ошибка:** оставить `table-layout: auto`.  
  **Почему:** содержимое будет менять ширины столбцов.  
  **Как избежать:** установить `table-layout: fixed` и задать ширины столбцов (как в `column-brand { width: 150px; }`).

- **Ошибка:** полагаться на `height` у `.data-row`, но не менять стили `mat-form-field`.  
  **Почему:** `mat-form-field` имеет внутренние wrapper/label, которые всё равно увеличат высоту.  
  **Как избежать:** явно задать `height:100%` для `.mat-form-field` и сделать infix flex‑контейнером.

- **Ошибка:** показывать `mat-error` внутри строки (увеличивает высоту).  
  **Как избежать:** показывать ошибки в другом месте (tooltip, snackbar) или позиционировать `mat-error` абсолютным образом, чтобы он не менял высоту строки.

- **Ошибка:** задавать `height` инпутам в `px`, не учитывая padding/box-sizing.  
  **Как избежать:** использовать `box-sizing: border-box` и рассчитывать высоту с учётом padding, либо использовать `height: 100%` для контейнера и меньшую фиксированную высоту для самого input.

- **Ошибка:** не тестировать на разных шрифтах/zoom.  
  **Как избежать:** проверять при разных масштабах и на мобильных — лучше использовать относительные единицы или гибкие вычисления (`calc()`).
